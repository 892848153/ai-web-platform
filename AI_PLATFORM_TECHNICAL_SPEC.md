# AI启航平台技术规格说明书

## 📋 项目概述

AI启航平台是一个面向职场人士的AI反焦虑一站式解决方案，提供AI工具广场、智能问答助手、最佳实践分享、AI学习中心和需求悬赏等功能模块。

### 🎯 核心目标
- 降低AI使用门槛，帮助用户快速上手AI工具
- 提供高质量的最佳实践和提示词模板
- 构建AI学习路径，从入门到精通
- 连接AI专家与需求方，促进知识变现

## 🏗️ 技术架构

### 前端架构

#### 核心框架
- **React 18**: 现代化前端框架，提供组件化开发体验
- **TypeScript**: 类型安全的JavaScript超集，提升代码质量
- **Vite**: 下一代前端构建工具，极速的开发体验
- **React Router DOM**: 客户端路由管理

#### 状态管理
- **Zustand**: 轻量级状态管理库，替代Redux
- **React Hooks**: 函数组件状态管理

#### 样式系统
- **Tailwind CSS**: 实用优先的CSS框架
- **自定义主题**: 赛博朋克风格暗色主题
- **Framer Motion**: 动画库，提供流畅的交互体验

#### UI组件
- **Lucide React**: 现代化图标库
- **自定义组件**: 基于业务需求开发的专用组件
- **响应式设计**: 移动端优先的响应式布局

### 后端架构

#### 数据库服务
- **Supabase**: 开源Firebase替代方案
  - PostgreSQL数据库
  - 实时数据同步
  - 行级安全策略(RLS)
  - 自动API生成

#### 服务器less函数
- **Supabase Edge Functions**: TypeScript编写的云函数
- **部署区域**: 全球多个区域部署，低延迟

#### 存储服务
- **Supabase Storage**: 文件存储服务
- **CDN加速**: 全球内容分发网络

## 🗄️ 数据架构

### 数据库设计

#### best_practices 表
```sql
CREATE TABLE public.best_practices (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  author TEXT NOT NULL,
  views INT DEFAULT 0,
  likes INT DEFAULT 0,
  tags TEXT[] DEFAULT '{}',
  content TEXT NOT NULL,
  prompt TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

#### demands 表（需求悬赏）
```sql
CREATE TABLE public.demands (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  budget TEXT,
  deadline DATE,
  status TEXT DEFAULT 'open',
  author_id UUID,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

#### workflows 表（工作流）
```sql
CREATE TABLE public.workflows (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL,
  description TEXT,
  steps JSONB,
  author_id UUID,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

### 数据流设计

#### 前端数据流
```
用户界面 → React组件 → Zustand状态管理 → Supabase客户端 → PostgreSQL
                                                      ↓
                                               Supabase Edge Functions → 通义千问API
```

#### 后端数据流
```
通义千问API → Supabase Edge Functions → 数据处理 → PostgreSQL → 前端展示
```

## 🔐 安全架构

### 认证授权
- **Supabase Auth**: 用户认证系统
- **行级安全(RLS)**: 数据库级别的访问控制
- **API密钥管理**: 环境变量存储敏感信息

### 数据安全
- **HTTPS加密**: 全站SSL加密
- **数据验证**: 前端 + 后端双重验证
- **XSS防护**: React自动转义 + 手动过滤
- **CSRF防护**: Supabase内置防护

### API安全
- **Edge Function代理**: 隐藏AI API密钥
- **请求限制**: 频率限制和用量控制
- **输入验证**: 严格的参数验证
